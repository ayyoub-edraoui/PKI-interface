Simulation d'utilisation d'OpenSSL

Dans cette page, nous allons simuler l'utilisation d'OpenSSL pour générer une paire de clés privées/publiques, créer un certificat auto-signé et signer un certificat d'une autorité de certification (CA).

Génération de clés privées/publiques

Pour générer une paire de clés privées/publiques, nous allons utiliser la commande suivante :

openssl genpkey -algorithm RSA -out private_key.pem -aes256

Cette commande va générer une clé privée RSA de 2048 bits et la stocker dans un fichier nommé private_key.pem. Le paramètre -aes256 permet de chiffrer la clé privée avec une phrase de passe.

Création d'un certificat auto-signé

Pour créer un certificat auto-signé, nous allons utiliser la commande suivante :

openssl req -x509 -new -key private_key.pem -out certificate.pem

Cette commande va générer un certificat auto-signé et le stocker dans un fichier nommé certificate.pem. La clé privée utilisée pour signer le certificat est celle générée précédemment.

Signature d'un certificat avec une autorité de certification (CA)

Pour signer un certificat avec une autorité de certification (CA), nous allons utiliser la commande suivante :

openssl x509 -req -in certificate.csr -CA ca_certificate.pem -CAkey ca_private_key.pem -CAcreateserial -out signed_certificate.pem

Cette commande va signer un certificat en utilisant la clé privée de l'autorité de certification (CA) et le certificat de l'autorité de certification (CA). Le certificat à signer est spécifié avec le paramètre -req, et le certificat signé est stocké dans un fichier nommé signed_certificate.pem.
